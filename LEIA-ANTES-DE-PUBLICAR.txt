ALFA FINANCEIRO V54 - REVISÃO DE BOTÕES, CLIENTES E SEGURANÇA

PUBLICAÇÃO
1. Substitua todos os arquivos do site pela pasta desta versão.
2. Publique principalmente o novo index.html.
3. Publique também o alfa-logo.png e alfa-logo.jpg otimizados.
4. No Firebase Console, publique o conteúdo de FIRESTORE-RULES-ALFA-FINANCEIRO.txt em Firestore Database > Rules.
5. Abra o site e pressione Ctrl + F5.

O QUE FOI CORRIGIDO
- Botão Editar cadastro do cliente com abertura reforçada.
- Botão Baixar honorário com checagem de permissão.
- Novo cliente, novo lançamento, nova receita e nova despesa com checagem de permissão.
- Layout de clientes ajustado para não depender de rolagem horizontal.
- Barra de rolagem vertical mais visível.
- Perfil Funcionário limitado incluído.
- Logs locais de auditoria para cliente, lançamento e baixa.
- Regras Firestore revisadas para restringir gravações.
- Logos otimizados para carregar mais rápido.

TESTE RÁPIDO
1. Entre como administrador.
2. Aba Clientes > Editar cadastro.
3. Salve uma alteração simples.
4. Baixe uma parcela em aberto.
5. Crie um novo cliente.
6. Crie uma nova receita.
7. Crie uma nova despesa.
8. Acesse Usuários e teste perfil Consulta ou Funcionário limitado.

DIAGNÓSTICO
No console do navegador, execute:
alfaDiagnosticoBotoes()

Se o site parecer antigo depois da publicação, use Ctrl + F5 ou teste em aba anônima.

V55:
- Correção reforçada do botão Editar cadastro na aba Clientes.
- Se o usuário não tiver permissão de edição, o sistema mostra alerta claro informando o e-mail logado.
